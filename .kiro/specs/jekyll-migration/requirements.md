# Документ требований

## Введение

Данный документ описывает требования к миграции существующего сайта-визитки с React/Vite на Jekyll для размещения на GitHub Pages. Сайт представляет собой одностраничное приложение с информацией об эксперте по формированию команд бизнес-аналитиков.

## Глоссарий

- **Система**: Jekyll-сайт - статический сайт, сгенерированный с помощью генератора статических сайтов Jekyll
- **React-сайт**: Исходный сайт, построенный на React, Vite и TypeScript
- **GitHub Pages**: Сервис хостинга статических сайтов от GitHub
- **Секция Hero**: Главная секция сайта с фотографией профиля, заголовком и кнопками контактов
- **Секция About**: Секция с описанием услуг и компетенций
- **Секция Footer**: Нижняя часть сайта с информацией об авторских правах
- **Контент**: Текстовая и графическая информация, отображаемая на сайте
- **Стили**: Визуальное оформление сайта (цвета, шрифты, отступы, анимации)
- **Конфигурация Jekyll**: Файл _config.yml с настройками генератора Jekyll

## Требования

### Требование 1

**Пользовательская история:** Как владелец сайта, я хочу мигрировать сайт на Jekyll, чтобы разместить его на GitHub Pages без необходимости сборки на стороне клиента

#### Критерии приемки

1. THE Jekyll-сайт SHALL генерировать статический HTML с идентичной структурой страницы React-сайту
2. THE Jekyll-сайт SHALL включать файл _config.yml с базовыми настройками для GitHub Pages
3. THE Jekyll-сайт SHALL использовать layout-файлы для структурирования HTML-разметки
4. THE Jekyll-сайт SHALL размещаться в корневой директории проекта для совместимости с GitHub Pages
5. THE Jekyll-сайт SHALL включать файл Gemfile с необходимыми зависимостями Jekyll

### Требование 2

**Пользовательская история:** Как посетитель сайта, я хочу видеть идентичный контент и структуру, чтобы получить ту же информацию, что и на React-сайте

#### Критерии приемки

1. THE Jekyll-сайт SHALL отображать секцию Hero с фотографией профиля, заголовком и описанием
2. THE Jekyll-сайт SHALL отображать кнопки контактов (Email, Telegram, GitHub, Резюме) с корректными ссылками
3. THE Jekyll-сайт SHALL отображать секцию About с тремя карточками услуг
4. THE Jekyll-сайт SHALL отображать секцию Footer с информацией об авторских правах и текущим годом
5. THE Jekyll-сайт SHALL включать все текстовые материалы на русском языке из React-сайта

### Требование 3

**Пользовательская история:** Как посетитель сайта, я хочу видеть идентичное визуальное оформление, чтобы сайт выглядел профессионально и привлекательно

#### Критерии приемки

1. THE Jekyll-сайт SHALL применять стили, визуально идентичные React-сайту
2. THE Jekyll-сайт SHALL использовать градиентный фон для секции Hero
3. THE Jekyll-сайт SHALL применять эффекты тени и hover-состояния для карточек и кнопок
4. THE Jekyll-сайт SHALL использовать адаптивный дизайн для корректного отображения на мобильных устройствах
5. THE Jekyll-сайт SHALL включать иконки Lucide или аналогичные для визуального оформления

### Требование 4

**Пользовательская история:** Как владелец сайта, я хочу легко обновлять контент, чтобы поддерживать актуальность информации без изменения кода

#### Критерии приемки

1. THE Jekyll-сайт SHALL хранить контент секции Hero в файле данных YAML или front matter
2. THE Jekyll-сайт SHALL хранить список услуг секции About в файле данных YAML
3. THE Jekyll-сайт SHALL использовать переменные Jekyll для вывода динамического контента (например, текущего года)
4. THE Jekyll-сайт SHALL позволять обновление контактных ссылок через конфигурационный файл
5. THE Jekyll-сайт SHALL включать комментарии в файлах данных для упрощения редактирования

### Требование 5

**Пользовательская история:** Как владелец сайта, я хочу разместить сайт на GitHub Pages, чтобы обеспечить бесплатный и надежный хостинг

#### Критерии приемки

1. THE Jekyll-сайт SHALL включать файл .gitignore с исключением директории _site и служебных файлов
2. THE Jekyll-сайт SHALL включать файл README.md с инструкциями по локальной разработке и деплою
3. THE Jekyll-сайт SHALL использовать структуру директорий, совместимую с GitHub Pages
4. THE Jekyll-сайт SHALL включать файл конфигурации для корректной работы с baseurl на GitHub Pages
5. THE Jekyll-сайт SHALL копировать статические ресурсы (изображения, PDF) в соответствующие директории

### Требование 6

**Пользовательская история:** Как разработчик, я хочу удалить зависимости React, чтобы упростить проект и уменьшить его размер

#### Критерии приемки

1. WHEN миграция завершена, THE Jekyll-сайт SHALL не содержать файлов конфигурации React (package.json, vite.config.ts, tsconfig.json)
2. WHEN миграция завершена, THE Jekyll-сайт SHALL не содержать директории src с React-компонентами
3. WHEN миграция завершена, THE Jekyll-сайт SHALL не содержать директории node_modules
4. THE Jekyll-сайт SHALL использовать только Ruby-зависимости для генерации статического сайта
5. THE Jekyll-сайт SHALL иметь размер репозитория меньше, чем React-сайт (без node_modules)
