# Дизайн-документ: Миграция сайта-визитки на Jekyll

## Обзор

Данный документ описывает архитектуру и дизайн миграции React-сайта на Jekyll для размещения на GitHub Pages. Миграция предполагает полное преобразование динамического React-приложения в статический сайт с сохранением всего функционала, контента и визуального оформления.

## Архитектура

### Структура проекта Jekyll

```
/
├── _config.yml                 # Конфигурация Jekyll
├── _data/
│   ├── hero.yml               # Данные секции Hero
│   ├── about.yml              # Данные секции About
│   └── contacts.yml           # Контактные ссылки
├── _layouts/
│   └── default.html           # Основной layout
├── _includes/
│   ├── head.html              # HTML head с мета-тегами
│   ├── hero.html              # Секция Hero
│   ├── about.html             # Секция About
│   └── footer.html            # Секция Footer
├── assets/
│   ├── css/
│   │   └── main.css           # Основные стили
│   ├── images/
│   │   └── profile-photo.jpg  # Фото профиля
│   └── files/
│       └── resume.pdf         # Резюме
├── index.html                 # Главная страница
├── Gemfile                    # Ruby зависимости
├── .gitignore                 # Игнорируемые файлы
└── README.md                  # Документация
```

### Принципы архитектуры

1. **Разделение контента и представления**: Контент хранится в YAML-файлах, HTML-шаблоны отвечают за отображение
2. **Модульность**: Каждая секция сайта выделена в отдельный include-файл
3. **Переиспользование**: Layout используется для общей структуры страницы
4. **Статичность**: Весь контент генерируется на этапе сборки Jekyll

## Компоненты и интерфейсы

### 1. Конфигурация Jekyll (_config.yml)

**Назначение**: Основные настройки сайта для Jekyll и GitHub Pages

**Содержимое**:
```yaml
title: "Эксперт по формированию команд бизнес-аналитиков"
description: "Фанат AI инструментов. Помогаю компаниям создавать эффективные команды аналитиков"
baseurl: ""
url: "https://yourusername.github.io"

# Build settings
markdown: kramdown
theme: null
plugins:
  - jekyll-seo-tag

# Exclude from processing
exclude:
  - Gemfile
  - Gemfile.lock
  - node_modules
  - vendor
```

### 2. Файлы данных (_data/)

#### hero.yml
**Назначение**: Хранение контента секции Hero

**Структура**:
```yaml
title: "Эксперт по формированию команд"
subtitle: "бизнес-аналитиков"
description: "Фанат AI инструментов. Помогаю компаниям создавать эффективные команды аналитиков и внедрять современные технологии."
profile_image: "/assets/images/profile-photo.jpg"
```

#### about.yml
**Назначение**: Хранение списка услуг для секции About

**Структура**:
```yaml
title: "О моей работе"
description: "Специализируюсь на построении эффективных команд бизнес-аналитиков и интеграции современных AI инструментов в рабочие процессы."
features:
  - icon: "users"
    title: "Формирование команд"
    description: "Создаю сильные команды бизнес-аналитиков с фокусом на синергию и эффективность"
  - icon: "sparkles"
    title: "AI инструменты"
    description: "Внедряю передовые AI решения для автоматизации аналитических процессов"
  - icon: "target"
    title: "Стратегический подход"
    description: "Разрабатываю долгосрочные стратегии развития аналитических команд"
```

#### contacts.yml
**Назначение**: Хранение контактных ссылок

**Структура**:
```yaml
email: "your.email@example.com"
telegram: "https://t.me/yourusername"
github: "https://github.com/yourusername"
resume: "/assets/files/resume.pdf"
```

### 3. Layout (_layouts/default.html)

**Назначение**: Основная структура HTML-страницы

**Компоненты**:
- HTML5 doctype и структура
- Подключение стилей
- Мета-теги для SEO
- Контейнер для контента страницы

**Интерфейс**:
- Использует `{% include %}` для подключения head.html
- Использует `{{ content }}` для вставки содержимого страницы

### 4. Includes (_includes/)

#### head.html
**Назначение**: HTML head с мета-тегами и подключением стилей

**Содержит**:
- Мета-теги charset, viewport
- Title и description из _config.yml
- Подключение CSS
- SEO-теги

#### hero.html
**Назначение**: Секция Hero с фото, заголовком и кнопками

**Данные**: Использует `site.data.hero` и `site.data.contacts`

**Структура**:
- Контейнер с градиентным фоном
- Изображение профиля с эффектом свечения
- Заголовок и подзаголовок
- Описание
- Кнопки контактов (Email, Telegram, GitHub, Резюме)

#### about.html
**Назначение**: Секция About с карточками услуг

**Данные**: Использует `site.data.about`

**Структура**:
- Заголовок и описание секции
- Grid-сетка с карточками
- Цикл `{% for feature in site.data.about.features %}` для генерации карточек
- Иконки, заголовки и описания услуг

#### footer.html
**Назначение**: Нижняя часть сайта

**Структура**:
- Копирайт с динамическим годом `{{ 'now' | date: "%Y" }}`
- Текст об авторских правах

### 5. Главная страница (index.html)

**Назначение**: Точка входа на сайт

**Структура**:
```yaml
---
layout: default
---

{% include hero.html %}
{% include about.html %}
{% include footer.html %}
```

## Стили (assets/css/main.css)

### Подход к стилизации

Используем чистый CSS с переменными для воссоздания дизайна React-сайта, который использовал Tailwind CSS.

### CSS-переменные

```css
:root {
  /* Цвета */
  --color-primary: #9b87f5;
  --color-background: #1A1F2C;
  --color-foreground: #FFFFFF;
  --color-muted: #8E9196;
  --color-card: #221F26;
  --color-border: #403E43;
  
  /* Градиенты */
  --gradient-hero: linear-gradient(to bottom, #1A1F2C, #2A1F3D);
  
  /* Тени */
  --shadow-card: 0 4px 6px -1px rgba(0, 0, 0, 0.1);
  --shadow-lg: 0 10px 15px -3px rgba(0, 0, 0, 0.1);
  
  /* Шрифты */
  --font-sans: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
}
```

### Основные классы

1. **Layout классы**: `.container`, `.section`, `.grid`
2. **Компонентные классы**: `.hero`, `.about`, `.footer`, `.card`, `.button`
3. **Утилитарные классы**: `.text-center`, `.mb-*`, `.p-*`
4. **Hover-эффекты**: Transitions для кнопок и карточек

### Адаптивность

Используем media queries для адаптации под разные размеры экранов:
- Mobile: < 768px
- Tablet: 768px - 1024px
- Desktop: > 1024px

## Иконки

### Решение для иконок

Вместо Lucide React используем одно из решений:

**Вариант 1: Font Awesome (рекомендуется)**
- Подключение через CDN
- Простая интеграция
- Большая библиотека иконок

**Вариант 2: Inline SVG**
- Встраивание SVG-кода напрямую в HTML
- Полный контроль над стилями
- Нет внешних зависимостей

**Выбранное решение**: Font Awesome через CDN для простоты и скорости разработки

### Маппинг иконок

- `Users` (Lucide) → `fa-users` (Font Awesome)
- `Sparkles` (Lucide) → `fa-sparkles` (Font Awesome)
- `Target` (Lucide) → `fa-bullseye` (Font Awesome)
- `Mail` (Lucide) → `fa-envelope` (Font Awesome)
- `Github` (Lucide) → `fa-github` (Font Awesome)
- `FileText` (Lucide) → `fa-file-alt` (Font Awesome)

## Обработка ошибок

### Отсутствующие данные

**Проблема**: YAML-файлы могут быть пустыми или содержать неполные данные

**Решение**: Использование условий Liquid для проверки наличия данных

```liquid
{% if site.data.hero.title %}
  <h1>{{ site.data.hero.title }}</h1>
{% else %}
  <h1>Заголовок не указан</h1>
{% endif %}
```

### Отсутствующие изображения

**Проблема**: Изображение профиля может отсутствовать

**Решение**: 
1. Использование placeholder-изображения
2. Проверка наличия файла перед отображением
3. Alt-текст для доступности

### Некорректные ссылки

**Проблема**: Контактные ссылки могут быть невалидными

**Решение**:
1. Валидация URL в YAML-комментариях
2. Использование `rel="noopener noreferrer"` для внешних ссылок
3. Проверка mailto: формата для email

## Стратегия тестирования

### Локальное тестирование

1. **Установка Jekyll**: `bundle install`
2. **Запуск локального сервера**: `bundle exec jekyll serve`
3. **Проверка в браузере**: `http://localhost:4000`

### Тестирование контента

- Проверка отображения всех секций
- Проверка корректности текстов на русском языке
- Проверка работы всех ссылок
- Проверка отображения изображений

### Тестирование стилей

- Визуальное сравнение с React-сайтом
- Проверка адаптивности на разных устройствах
- Проверка hover-эффектов
- Проверка градиентов и теней

### Тестирование на GitHub Pages

1. **Push в репозиторий**: Отправка кода в GitHub
2. **Настройка GitHub Pages**: Включение в Settings → Pages
3. **Проверка деплоя**: Проверка сайта по URL GitHub Pages
4. **Проверка baseurl**: Корректность путей к ресурсам

### Кросс-браузерное тестирование

- Chrome/Edge (Chromium)
- Firefox
- Safari (macOS/iOS)
- Мобильные браузеры

### Тестирование производительности

- Проверка времени загрузки страницы
- Проверка размера CSS-файла
- Проверка оптимизации изображений

### Этап 1: Подготовка

## Решения по дизайну

### Почему Jekyll?

- **Нативная поддержка GitHub Pages**: Не требуется CI/CD для сборки
- **Простота**: Меньше зависимостей и конфигурации
- **Производительность**: Статические файлы загружаются быстрее
- **SEO**: Лучшая индексация статического контента

### Почему YAML для данных?

- **Читаемость**: Легко редактировать без знания HTML
- **Структурированность**: Четкая организация данных
- **Переиспользование**: Данные можно использовать в разных местах

### Почему чистый CSS вместо Tailwind?

- **Простота**: Не требуется сборка или PostCSS
- **Контроль**: Полный контроль над стилями
- **Размер**: Меньший размер CSS-файла
- **Совместимость**: Работает везде без дополнительных инструментов

### Почему Font Awesome?

- **Доступность**: CDN-подключение без установки
- **Совместимость**: Аналоги для всех иконок Lucide
- **Надежность**: Проверенное решение с хорошей поддержкой

## Ограничения и компромиссы

### Ограничения

1. **Нет интерактивности**: Jekyll генерирует статические страницы, динамические функции требуют JavaScript
2. **Обновление контента**: Требуется commit и push для обновления (нет CMS)
3. **Время сборки**: GitHub Pages пересобирает сайт при каждом push

### Компромиссы

1. **Иконки**: Используем Font Awesome вместо Lucide (визуально похожи)
2. **Стили**: Воссоздаем Tailwind-стили вручную (может быть небольшая разница)
3. **Компоненты**: Нет переиспользуемых React-компонентов (используем includes)
